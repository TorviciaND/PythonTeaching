Introduction to Data Types

Python was originally developed as a general purpose programming language. The functionality to easily perform data analysis tasks was added later via the Numpy, Scipy, Pandas, Statsmodels, and Scikit-Learn libraries among others. In order to effectively use the libraries for data analysis and other purposes, it is important to understand some of the basic constructs of the language. The goal of this notebook is to help you understand the basic data types (type synonymous with object) available in Python.

The basic data types, also known a scalar types are:

  • str - strings
  • int - integers (whole numbers, no decimals)
  • float - floating point numbers (have decimal)
  • bool - boolean values either (True/False)
  • None - the python ‘null’ value

The following basic data type objects exist but are not important for our course

  • bytes - raw ASCII bytes
  • complex - complex number type
  • long - arbitrarily long int

What if you want to calculate \(\sqrt{3}\), \(ln(5)\) or \(e^{1.4}\)?

These function exist in the math library, which is part of base python. We need to import the math library to make those functions available to us. There are a couple of main ways to do this.

Importing Python Libraries

  • import math - if we import the math library this way, all of the functions in the math library are available to us as: math.<function_name>()

  • import math as m - if we import the math library this way, the functions in the math library are available to us as m.<function_name()>.

Note

We could have chosen any alias for our library. Here I chose to use m to represent math. Usually the alias is chosen to be short to reduce typing if the library name is long. Some libraries have a default abbreviation that is commonly accepted among Python programmers.

If we only want to import a specific function from the math library, we can import just that function. This avoids loading all the functions in the library and is very common when working with large libraries like sci-kit learn or tensorflow. For instance, if we only wanted the sqrt() function, we could type from math import sqrt. After importing the function this way, we can just type the function name without referencing any library.

Compound Boolean Expressions

Compound Boolean Expressions combine multiple Boolean values together to return a Boolean value (as opposed to the numeric value we saw above). We can create compound logicals with boolean operators:

  • not
  • or
  • and

not

not is the compliment operator. It will change a True Boolean value to False and a False Boolean value to True.

or

The or logical operator evaluates as an inclusive or. This means that as long as one of the Boolean inputs is True, then the entire expression evaluates to True.

Note

In the English language, we have inclusive or and exclusive or.

  • An inclusive or is read as “you need one or the other or both”.
    For example: You need a ticket or 5 dollars to get into the club.

  • An exclusive or is read as “you need to make a decision”.
    For example: You can have cake or ice cream (but not both).

and

The and logical operator will evaluate to True, only if all of the inputs are True.

fstrings

fstrings are a new addition in Python 3 that is very useful. Often we have information contained in variables that we want to combine into sensible output. One nice thing about fstrings is that is does not matter if the variable type is an int, float, boolean, or string, it just works!

To create an fstring, just put the letter f in front of your string. Within the string use {} around your variable names. The values of the variables will replace the variable names in the output.
